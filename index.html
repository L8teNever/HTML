Datenschutzerklärung – CLEO Editor

Kurzfassung: CLEO ist ein reiner Browser-Editor. Deine Dateien, dein Code und deine Einstellungen verlassen deinen Browser nie ohne dein ausdrückliches Zutun. Es gibt keinen Account, keinen Backend-Server und keine Cloud-Speicherung.

Daten, die ausschließlich lokal in deinem Browser gespeichert werden Nur lokal

Folgende Daten werden über die localStorage-API deines Browsers auf deinem eigenen Gerät gespeichert. Sie verlassen deinen Browser niemals und werden nicht an Server übertragen:

  • Deine Dateien & Projekte – Sämtlicher Code (HTML, CSS, JS, etc.) und Dateinamen, die du im Editor erstellst oder hochlädst, werden ausschließlich im localStorage deines Browsers gespeichert (Schlüssel: codeplay.files). Kein Server sieht diese Daten.
  • Aktiver Tab, geöffnete Tabs, aktive Datei – Der Zustand deines Editors (welche Datei du gerade bearbeitest, welche Tabs offen sind) wird lokal unter codeplay.state gespeichert.
  • Einstellungen – Farbthema, Schriftgröße, Schriftfamilie, Zeilenhöhe, Minimap-Einstellung, Wortumbruch, Format-on-Save und UI-Schriftgröße werden lokal gespeichert (Schlüssel: cleo.settings.*).
  • Eigene Farben – Benutzerdefinierte Akzentfarben, Hintergrundfarben und Textfarben (Schlüssel: cleo.settings.custom*).
  • Onboarding-Status – Ob du den Willkommensbildschirm bereits gesehen hast (codeplay.onboarded).
  • Sprachauswahl – Deine gewählte Sprache (Deutsch / Englisch) unter codeplay.lang.
  • Leere Ordner – Von dir erstellte leere Ordnerstrukturen (codeplay.emptyFolders).
  • Live-Verbindungs-Metadaten – Name des zuletzt verbundenen lokalen Ordners (bei Verwendung der File System Access API), nur lokal.
Du kannst jederzeit alle gespeicherten Daten löschen, indem du in deinem Browser die Browserdaten für diese Seite löschst (Einstellungen → Datenschutz → Browserdaten löschen → Gespeicherte Daten / localStorage).

Daten & Netzwerkverbindungen (externe Ressourcen) Netzwerk

CLEO lädt beim Starten externe Bibliotheken und Schriften von Drittanbietern. Bei diesen Anfragen wird deine IP-Adresse und dein Browser-User-Agent an die jeweiligen Server übertragen – wie bei jedem normalen Webseitenaufruf. Dein Code oder deine Dateien werden dabei nicht übertragen.

  • Monaco Editor (CDN: jsdelivr.net) – Der Code-Editor basiert auf dem Monaco-Editor von Microsoft, der von cdn.jsdelivr.net geladen wird. Beim Laden sendet dein Browser eine HTTP-Anfrage mit IP und User-Agent an jsDelivr (CDN-Anbieter, via Cloudflare).
  • Google Fonts (fonts.googleapis.com, fonts.gstatic.com) – Die Schriftart „JetBrains Mono" wird von Google Fonts geladen. Google kann dabei IP-Adresse, Zeitstempel und Browser-Informationen erfassen. Weitere Informationen: policies.google.com/privacy.
  • html2canvas (cdnjs.cloudflare.com) – Nur wenn du den Screenshot-Designer öffnest, wird diese Bibliothek von Cloudflare's CDN geladen. Dabei werden IP und User-Agent an Cloudflare übertragen.
  • Prism.js & Prism-CSS (cdnjs.cloudflare.com) – Ebenfalls nur bei Screenshot-Nutzung werden Syntax-Highlighting-Bibliotheken von Cloudflare's CDN geladen.
  • Tailwind CSS CDN (cdn.tailwindcss.com) – Für den Willkommensbildschirm wird Tailwind CSS vom offiziellen CDN geladen. Dabei erfolgt eine Netzwerkanfrage an Tailwind's Server.
Hinweis zu Cloudflare Tunnel: Wenn du CLEO über einen Cloudflare Tunnel aufrufst (z. B. über eine öffentliche URL statt file://), läuft dein HTTP-Traffic durch Cloudflare's Infrastruktur. Cloudflare kann dabei Verbindungsmetadaten (IP-Adresse, Zeitstempel, Anfragepfade) sehen – jedoch keinen deiner Dateien oder localStorage-Inhalte, da diese ausschließlich im Browser verbleiben. Cloudflare verschlüsselt den Transit (TLS). Weitere Informationen: cloudflare.com/privacypolicy.

Was CLEO definitiv NICHT sammelt Keine Daten

  • ❌ Kein Tracking, keine Analyse-Tools (kein Google Analytics, kein Matomo, kein Plausible).
  • ❌ Keine Cookies (weder First-Party noch Third-Party-Cookies von CLEO selbst).
  • ❌ Kein Account, keine Registrierung, keine Anmeldedaten.
  • ❌ Dein Code, deine Dateien oder Dateinamen werden niemals zu einem Server hochgeladen.
  • ❌ Keine Telemetriedaten über deine Aktionen im Editor (welche Dateien du öffnest, was du tippst).
  • ❌ Keine Weitergabe von Daten an Dritte durch CLEO selbst.
  • ❌ Kein serverseitiges Session-Tracking.

Live-Verbindung mit lokalem Ordner (File System Access API)

Wenn du die Funktion „Live mit Ordner verbinden" oder „Live mit Dateien verbinden" verwendest (nur in Chrome/Edge verfügbar), greift CLEO mit deiner Erlaubnis direkt auf Dateien auf deiner Festplatte zu:

  • Lese-/Schreibzugriff nur auf explizit gewählte Dateien/Ordner – Du wählst selbst aus, welcher Ordner oder welche Dateien CLEO zugänglich gemacht werden. Der Zugriff endet, sobald du die Seite schließt oder neu lädst.
  • Keine Übertragung ans Netzwerk – Die Dateiinhalte werden nur lokal im Browser verarbeitet und direkt zurückgeschrieben. Sie verlassen deinen Computer nicht.
  • Berechtigung durch den Browser – Der Browser zeigt eine eigene Berechtigungsabfrage an, bevor CLEO auf dein Dateisystem zugreifen darf.

Screenshot-Designer

Wenn du den Screenshot-Designer verwendest, werden dynamisch externe Bibliotheken geladen:

  • html2canvas und Prism.js werden von cdnjs.cloudflare.com geladen – dabei werden IP und User-Agent an Cloudflare übertragen.
  • Das erzeugte Bild (PNG) wird ausschließlich lokal in deinem Browser generiert und direkt heruntergeladen. Es wird nirgendwo hochgeladen.
  • Dein Code, der im Screenshot zu sehen ist, bleibt vollständig auf deinem Gerät.

Deine Rechte & Datenlöschung

Da CLEO keine personenbezogenen Daten auf Servern speichert, bedarf es keiner formellen Löschanfrage bei uns. Du kannst alle gespeicherten Daten selbst löschen:

  • Projektdaten löschen: Klicke im Editor auf „Projekt zurücksetzen" oder lösche den localStorage deines Browsers für diese Seite.
  • Browser-Einstellungen: Über Browsereinstellungen → Datenschutz → Browserdaten löschen → Gespeicherte Websitedaten.
  • Drittanbieter-Datenschutzerklärungen:
    • jsDelivr: jsdelivr.com/privacy
    • Google Fonts / Google: policies.google.com/privacy
    • Cloudflare: cloudflare.com/privacypolicy

Änderungen dieser Erklärung

Da CLEO eine rein clientseitige Anwendung ohne Nutzerdatenbank ist, betreffen Änderungen dieser Erklärung ausschließlich etwaige neue externe Bibliotheken oder Dienste, die eingebunden werden. Bei wesentlichen Änderungen wird die Versionsnummer in der Statusleiste aktualisiert.
